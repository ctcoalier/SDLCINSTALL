Spec Driven Coding

Go from idea to a planned, implemented feature with GitHub Spec Kit and the Superpowers plugin — orchestrated end-to-end inside Claude Code.

Why spec-driven?

Spec-driven development front-loads the thinking: you write a clear spec, turn it into a plan, break the plan into tasks, and only then generate code — each artifact reviewable before the next. Spec Kit structures those artifacts with /speckit.* slash commands; Superpowers brings the process discipline — brainstorming the approach, writing the plan, and test-driven implementation. Used together, Spec Kit gives the skeleton and Superpowers keeps every step honest.

Prerequisites

Tooling

Claude Code

Installed and opened inside your project directory so it has your code as context.

Plugin

Superpowers

The Superpowers plugin installed, so skills like brainstorming and test-driven-development trigger by name.

CLI

uv / uvx

Astral's uv available, so the specify CLI can bootstrap Spec Kit into your repo.

Create a feature, start to finish

A real Claude Code session — Spec Kit commands interleaved with Superpowers skills. Swap "saved searches" for whatever you're building.

claude — spec-driven-coding 
$ uvx --from git+https://github.com/github/spec-kit.git specify init --here --ai claude
✔ Spec Kit initialized · slash commands installed for Claude Code

$ claude

> /speckit.constitution
  Establishing project principles — quality, testing, UX, performance…
  ✔ Wrote .specify/memory/constitution.md

> Let's add saved searches. Brainstorm the approach first.
  ⚡ superpowers:brainstorming — exploring intent, users, and edge cases
  ✔ Agreed scope: name a filter set, list saved searches, re-apply one

> /speckit.specify Saved searches: name and revisit a set of filters
  ✔ specs/001-saved-searches/spec.md  (user stories + requirements)

> /speckit.plan Use the existing stack; no new services
  ✔ plan.md · research.md · data-model.md · contracts/

> /speckit.tasks
  ✔ tasks.md — 14 ordered, dependency-aware tasks

> /speckit.implement
  ⚡ superpowers:test-driven-development — red → green per task
  ✔ 14/14 tasks complete · tests passing

The steps, in plain English

Walkthrough

The same flow as the terminal, readable top to bottom.

  1. Initialize Spec Kit in your repo with specify init --here --ai claude. It installs the /speckit.* slash commands for Claude Code.
  2. Set the constitution — /speckit.constitution records the project's principles (quality, testing, UX, performance) so every later step is judged against them.
  3. Brainstorm first — before specifying, let Superpowers' brainstorming skill pin down intent, users, and edge cases. Spec quality starts here.
  4. Specify — /speckit.specify turns the agreed scope into a spec with user stories and requirements.
  5. Plan — /speckit.plan adds the technical approach: architecture, data model, and contracts, constrained to your existing stack.
  6. Break into tasks — /speckit.tasks produces an ordered, dependency-aware task list.
  7. Implement — /speckit.implement executes the tasks. Pair it with Superpowers' test-driven-development so each task goes red → green before moving on.
